# FIFO Verification

Small SystemVerilog testbench and DUT for a 7-entry FIFO used for functional verification and simulation.

## Repository layout

- `rtl/fifo.sv` — FIFO Design (SystemVerilog)
- `tb/sv_tb.sv` — Testbench (SystemVerilog)
- `dump.vcd` — Example VCD generated by simulation

## Requirements

- EDA Playground (https://www.edaplayground.com) — recommended for quick online SystemVerilog simulation; alternatively use a local simulator such as QuestaSim / ModelSim or another tool that supports SystemVerilog.
- GTKWave (optional) to view the `dump.vcd` when running locally

## Run (QuestaSim / qrun)

From the repository root you can run the example simulation used during development:

```sh
# run from repo root
qrun -batch -timescale 1ns/1ns -mfcu rtl/fifo.sv tb/sv_tb.sv -do "run -all; exit"
```

After the run, open `dump.vcd` with GTKWave:

```sh
gtkwave dump.vcd
```

### Run in EDA Playground (web)

- Go to https://www.edaplayground.com
- Create two files (use the file list on the left): `rtl/fifo.sv` and `tb/sv_tb.sv`, and paste the corresponding source files into each.
- Select `SystemVerilog` as the language and choose a supported simulator (for example, `QuestaSim` if available on the site)
- Click `Run`. The console will show simulation output; use the waveform viewer on EDA Playground or download the VCD for local viewing.

## Notes and tips

- The testbench uses a small `class data` with `rand`/`randc` fields and calls `randomize()` each cycle to drive `push`/`pop`/`data_in`.

- The simulation log in `log.log` shows fifo contents and occasional `pass` messages when the DUT and the reference match.
